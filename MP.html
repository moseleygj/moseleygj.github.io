01

Install MicroPython on your Pico

    Step 1: Install MicroPython on your Pico

  • Press and hold the BOOTSEL button on your Pico.
  • Plug the Pico into your computer using a USB cable while still holding the button.
  • Release the button once connected. The Pico will appear on your computer as a flash drive named RPI-RP2.
  • Download the correct MicroPython .uf2 file for your specific board from the official Raspberry Pi Documentation page.
  • Drag and drop the .uf2 file into the RPI-RP2 drive. The Pico will automatically reboot and finish the setup.

Step 2: Use Thonny to write and load code

  • Download and open Thonny on your computer.
  • Select the interpreter: Click the Python version in the bottom-right corner of the Thonny window and choose MicroPython (Raspberry Pi Pico).
  • Write your program: Type or paste your code directly into the main editor window.
  • Save to the board: Click File > Save As and select Raspberry Pi Pico.
  • Set to auto-run (Optional): Name your file main.py if you want the code to execute automatically every time the Pico powers on.

02

Understanding the Basics

The raspberry Pi is a single Board Computer(SBC) AKA a miniturized self contained computer. On this computer there are pins that allow the user to interact with it. The ones of focus will be the GPIO pins. A GPIO is a physical connection pin on a computer board or microchip that can be programmed to send or receive electrical signals.

As an Input: The pin reads electrical voltage to sense if a button is pressed or if a sensor (like a motion detector or thermometer) is triggered.
As an Output: The pin sends out electrical voltage to turn on an LED, trigger a relay, or spin a motor.

02

Matrix Wiring & Logic Assembly

When you are building a custom keyboard using a microcontroller like the Raspberry Pi Pico, you are dealing with a limited number of GPIO (General Purpose Input/Output) pins.
The Raspberry Pi Pico has 26 usable GPIO pins. If you tried to connect every single key of a standard keyboard directly to its own pin, you would run out of pins almost immediately: A direct-pin setup for a small macro pad (12 keys) = 12 pins (Doable).

A direct-pin setup for a 60% keyboard (~61 keys) = 61 pins (Impossible on a Pico).
A keyboard matrix solves this by routing the keys into a grid of Rows and Columns, drastically reducing the number of pins required.

For example, to wire up a 60-key keyboard:

Without a Matrix: You need 60 pins (The Pico cannot do this).
With a Matrix (5 Rows × 12 Columns): 5×12=60 keys. You only need 17 pins (5+12).

Hardware Expandability

This setup leaves plenty of leftover GPIO pins on your Pico. You can use these extra pins to add features like RGB underglow, OLED displays, or rotary encoders.

Matrix & Layers

The keypad uses a 3x3 matrix (9 buttons total) mapped across 4 software layers:

  • How Layers Work: Toggling to a different layer reassigns the entire grid to a fresh stack of commands.
  • Button Count: You have 8 usable buttons per layer, because 1 button is always reserved to switch between layers (9 - 1 = 8).
  • Total Actions: This gives you a total of 32 macro keys packaged into a tiny 9-button grid.

import board
import digitalio
import keypad
import analogio
import usb_hid
import time
import pwmio
from adafruit_hid.consumer_control import ConsumerControl
from adafruit_hid.consumer_control_code import ConsumerControlCode as CCC
from adafruit_hid.keyboard import Keyboard
from adafruit_hid.keycode import Keycode

# -------------------- MACROPAD SETUP --------------------
rows = [board.GP13, board.GP14, board.GP15]
cols = [board.GP10, board.GP11, board.GP12]

keys = keypad.KeyMatrix(row_pins=rows, column_pins=cols, columns_to_anodes=True)

# -------------------- POTENTIOMETER SETUP --------------------
POT_PIN = board.GP26
SAMPLES_MEDIAN = 9
IIR_ALPHA = 0.30
VOLUME_LEVELS = 17
HYSTERESIS_STEPS = 1
SEND_COOLDOWN = 0.12
CONFIRM_COUNT = 3

pot = analogio.AnalogIn(POT_PIN)

def read_median(n=SAMPLES_MEDIAN, pause=0.001):
    vals = []
    for _ in range(n):
        vals.append(pot.value)
        time.sleep(pause)
    vals.sort()
    return vals[len(vals)//2]

min_adc = 5
max_adc = 65000

def adc_to_step(adc_val, min_adc=min_adc, max_adc=max_adc):
    if adc_val < min_adc:
        adc_val = min_adc
    if adc_val > max_adc:
        adc_val = max_adc
    return int((adc_val - min_adc) / (max_adc - min_adc) * VOLUME_LEVELS)

# -------------------- HID SETUP --------------------
cc = ConsumerControl(usb_hid.devices)
keyboard = Keyboard(usb_hid.devices)

# -------------------- LED LAYER INDICATORS --------------------
led_pins = [board.GP16, board.GP17, board.GP18, board.GP19]
leds = []

for pin in led_pins:
    leds.append(pwmio.PWMOut(pin, frequency=1000, duty_cycle=0))

current_layer = 0

def set_led(index, brightness):
    leds[index].duty_cycle = int(65535 * brightness)

def update_leds():
    for i in range(len(leds)):
        set_led(i, 1.0 if i == current_layer else 0.0)

def startup_animation():
    FADE_STEPS = 50
    FADE_DELAY = 0.013
    PAUSE_BETWEEN = 0.3

    for _ in range(2):
        for i in range(FADE_STEPS + 1):
            brightness = i / FADE_STEPS
            for led in leds:
                led.duty_cycle = int(65535 * brightness)
            time.sleep(FADE_DELAY)
        for i in range(FADE_STEPS, -1, -1):
            brightness = i / FADE_STEPS
            for led in leds:
                led.duty_cycle = int(65535 * brightness)
            time.sleep(FADE_DELAY)
        time.sleep(PAUSE_BETWEEN)

    set_led(current_layer, 1.0)
    time.sleep(0.3)
    set_led(current_layer, 0.0)
    time.sleep(0.3)
    update_leds()

# -------------------- INITIAL POT STATE --------------------
raw = read_median()
smoothed = raw
current_step = adc_to_step(smoothed)
last_send_time = 0.0
confirm_target = current_step
confirm_counter = CONFIRM_COUNT

print("Macropad + Volume Knob + Layer LEDs Ready")
startup_animation()

# -------------------- MAIN LOOP --------------------
while True:
    event = keys.events.get()
    if event and event.pressed:
        # Layer / media keys
        if event.key_number == 0:
            current_layer = (current_layer + 1) % 4
            update_leds()

        elif event.key_number == 1:  # Paste then Enter
            keyboard.press(Keycode.CONTROL, Keycode.V)
            keyboard.release_all()
            time.sleep(0.05)
            keyboard.press(Keycode.ENTER)
            keyboard.release_all()

        elif event.key_number == 2:  # Next track
            cc.send(CCC.SCAN_NEXT_TRACK)

        elif event.key_number == 3:  # Mute
            cc.send(CCC.MUTE)

        elif event.key_number == 4:  # Paste only
            keyboard.press(Keycode.CONTROL, Keycode.V)
            keyboard.release_all()

        elif event.key_number == 5:  # Play/Pause
            cc.send(CCC.PLAY_PAUSE)

        elif event.key_number == 6:  # Mute / other
            cc.send(CCC.MUTE)

        elif event.key_number == 7:  # Copy
            keyboard.press(Keycode.CONTROL, Keycode.C)
            keyboard.release_all()

        elif event.key_number == 8:  # Back / previous
            cc.send(CCC.SCAN_PREVIOUS_TRACK)

    # Volume knob
    raw = read_median()
    smoothed = int((1.0 - IIR_ALPHA) * smoothed + IIR_ALPHA * raw)
    target_step = adc_to_step(smoothed)

    if target_step == confirm_target:
        if confirm_counter < CONFIRM_COUNT:
            confirm_counter += 1
    else:
        confirm_target = target_step
        confirm_counter = 1

    if confirm_counter >= CONFIRM_COUNT:
        diff = confirm_target - current_step
        if abs(diff) >= HYSTERESIS_STEPS:
            now = time.monotonic()
            if now - last_send_time >= SEND_COOLDOWN:
                if diff > 0:
                    cc.send(CCC.VOLUME_INCREMENT)
                    current_step += 1
                else:
                    cc.send(CCC.VOLUME_DECREMENT)
                    current_step -= 1
                current_step = max(0, min(VOLUME_LEVELS, current_step))
                last_send_time = now

    time.sleep(0.01)
